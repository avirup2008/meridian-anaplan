# Requirements: Meridian v2.0

**Defined:** 2026-05-10
**Core Value:** Anaplan model builders get instant, AI-powered analysis of their entire model without leaving the browser.

## v2.0 Requirements

### Infrastructure

- [ ] **INFRA-01**: Developer can add section structure comments to index.html dividing it into Connect, Picker, Fetch, and Dashboard sections before any new feature code lands
- [ ] **INFRA-02**: Project has @anthropic-ai/sdk, @vercel/blob, and pdfmake declared in package.json with pinned versions
- [ ] **INFRA-03**: vercel.json has a functions{} block with explicit maxDuration declared per endpoint (60s for blueprint and analyze, 30s for share, 10s for connect and models)
- [ ] **INFRA-04**: api/generate.js uses Claude Haiku via @anthropic-ai/sdk instead of Gemini 2.0 Flash, replacing GEMINI_API_KEY with ANTHROPIC_API_KEY

### Connection

- [ ] **CONN-01**: User can enter Anaplan username and password, submit to POST /api/connect, and see a confirmation card showing their workspace name and model count on success
- [ ] **CONN-02**: User can browse their Anaplan workspaces and models (grouped by workspace, with recently used models surfaced at top) and select one to proceed to blueprint fetch
- [ ] **CONN-03**: User who has been connected for ~35 minutes sees a re-auth prompt rather than a confusing API error when their Anaplan session token expires mid-session
- [ ] **CONN-04**: User can upload a CSV blueprint file as a fallback when live Anaplan connection is unavailable, and receive single-module Claude Haiku analysis

### Blueprint

- [ ] **BPRT-01**: System fetches all modules and all line items for the selected model server-side via POST /api/blueprint, batching 20 modules at a time in parallel
- [ ] **BPRT-02**: User sees live module and line item counts updating in the browser as blueprint data arrives (SSE via fetch + ReadableStream, not polling)
- [ ] **BPRT-03**: Blueprint JSON is written to Vercel Blob server-side and its URL is passed to /api/analyze — raw blueprint JSON never passes through a function request body
- [ ] **BPRT-04**: System catches Anaplan 429 rate limit errors with 10-second backoff, skips modules that fail after retry, and surfaces a partial-load warning to the user rather than failing the entire fetch

### Analysis

- [ ] **ANLZ-01**: User sees a model health score (0–100), a tiered verdict (Good / Needs Work / Critical), an executive summary paragraph, and 5 dimension scores (Structure, Formula, Best Practice, Naming, Performance) generated by Claude Sonnet
- [ ] **ANLZ-02**: User sees improvement suggestions grouped by domain (Structural / Formula / Best Practice / Naming) with a triage tag (Fix Now / Consider / Monitor) per suggestion, each showing reasoning and a recommended action, generated by Claude Haiku per module in parallel
- [ ] **ANLZ-03**: Server performs an extraction pass on the blueprint JSON to produce a structured summary (module names, line item counts, formula patterns, dependency graph) before sending to Claude — ensuring input fits within Haiku's 200K token limit
- [ ] **ANLZ-04**: User can view a cross-module data flow story generated by Claude Sonnet, and click any module node or inline module name to drill into a per-module note showing purpose, receives-from, sends-to, and risks

### Export & Share

- [ ] **EXPRT-01**: User can toggle which sections to include in the export (verdict, suggestions, notes) and see a live PDF preview update in the right panel as they change toggles
- [ ] **EXPRT-02**: User can download a multi-section PDF generated by pdfmake with proper pagination, selectable text, Meridian branding, and the sections they selected
- [ ] **EXPRT-03**: User can generate a shareable read-only link via POST /api/share that stores a report snapshot in Vercel Blob and returns a URL valid for 7 days — accessible to recipients without a login
- [ ] **EXPRT-04**: System automatically deletes expired Vercel Blob reports after 7 days via a Vercel Cron job that calls del() on blobs whose encoded expiry timestamp has passed

### UI Overhaul

- [ ] **UI-01**: App has 5 distinct screens — Connect, Model Picker, Blueprint Fetch, Dashboard, and Shared Report view — replacing the v1 single upload-centric layout
- [ ] **UI-02**: Dashboard has 4 panel tabs (Verdict / Suggestions / Notes / Export) with UI state preserved when switching between tabs (scroll position, filter selections, etc.)
- [ ] **UI-03**: Suggestions panel shows a filter bar with triage pills (All / Fix Now / Consider / Monitor) and domain pills, with suggestion cards grouped under collapsible domain headers, each showing a triage badge, reasoning paragraph, and action box
- [ ] **UI-04**: Notes panel shows a cross-module story view with clickable flow diagram nodes and inline module name links that drill into per-module notes with a breadcrumb back to the story

## Future Requirements

### Performance & Scale

- **PERF-01**: Support for Anaplan models with 200+ modules (current batch approach may hit timeout at scale)
- **PERF-02**: Incremental re-analysis of individual modules without refetching full blueprint

### Auth

- **AUTH-01**: OAuth 2.0 Client Credentials flow as alternative to Basic Auth (Anaplan supports this for service accounts)

### Collaboration

- **COLLAB-01**: Shareable links with comment threads for team review
- **COLLAB-02**: Report versioning — compare two analysis runs on same model over time

## Out of Scope

| Feature | Reason |
|---------|--------|
| Anaplan write-back | Read-only integration only — modifying models via API is out of scope for v2 |
| Real-time collaboration on shared links | Static snapshots via Blob; live multi-user editing adds significant complexity |
| User accounts / persistent storage | No database; Blob handles sharing without auth infrastructure |
| Mobile layout | Desktop browser only — Anaplan builders work on desktop |
| OAuth in v2.0 | Basic Auth is sufficient; OAuth adds setup complexity for users |
| i18n | English only for v2 |
| Video/GIF export | Not a format Anaplan teams use for handover |
| Puppeteer for PDF | Chromium binary (~150MB) exceeds Vercel's 50MB function bundle limit |
