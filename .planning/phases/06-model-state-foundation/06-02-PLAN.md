<objective>
Implement `api/model-state.js` — the v3.0 replacement for `api/blueprint.js`. The endpoint:
  1. Authenticates against Anaplan via Basic Auth → AnaplanAuthToken (D-01, copy from blueprint.js).
  2. Issues two PARALLEL fetches (`/modules` + `/lineItems?includeAll=true`) in a single round-trip — no per-module batching loop (MSF-01, D-01).
  3. Joins line items to modules by the field name documented in `06-SPIKE-RESULT.md` (D-02).
  4. Excludes decorator/separator modules using `isDecorativeModuleName()` from `analysis-core.js` (MSF-04).
  5. Serializes the functional modules into a compact, tab-separated, line-per-item text format with formula truncation at 150 chars (MSF-02).
  6. Computes an evidence pack with four admissibility gates: fetchCompleteness, formulaCoverage, graphDensity, namingCoverage + `blockedConclusions` array (MSF-03).
  7. Writes the compact text to Vercel Blob under the `model-state/` prefix (D-05).
  8. Streams SSE stage events: `auth → loading → serializing → writing → complete` (D-08).

Purpose: This is the data foundation for all v3.0 intelligence (comprehension, health, chat, build). Every downstream endpoint reads the Blob and evidence pack produced here.

Output: `api/model-state.js` — a single file, ~350-450 lines, implementing all of MSF-01, MSF-02, MSF-03, MSF-04.

    **Helper 1: `serializeModelState(functionalModules)` → string**

    Produces a compact text format. Algorithm:
    ```
    For each module in functionalModules:
      Output:  MODULE\t{id}\t{name}\t{prefixOrEmpty}\n
      For each lineItem li in module.lineItems:
        Determine type label:
          - If li.formula is a non-empty string → "CALC"

    Return the joined string.
    ```

    ```javascript
    const withFormulas = functional.filter(m => m.lineItems.some(li => typeof li.formula === 'string' && li.formula.trim().length > 0));
    const withNames = functional.filter(m => DISCO_PREFIX_REGEX.test(m.name));
    const totalEdges = computeDependencyEdges(functional);

    const fetchCompleteness = totalModuleCount > 0 ? fetchedModuleIds.size / totalModuleCount : 0;
    const formulaCoverage   = functional.length > 0 ? withFormulas.length / functional.length : 0;
    const graphDensity      = functional.length > 1 ? Math.min(1, totalEdges / (functional.length * 2)) : 0;
    const namingCoverage    = functional.length > 0 ? withNames.length / functional.length : 0;

    const blockedConclusions = [];
    if (fetchCompleteness < GATE_THRESHOLDS.fetchCompleteness) {
      blockedConclusions.push(`Architecture and health claims suppressed — fetch completeness ${fetchCompleteness.toFixed(2)} (minimum ${GATE_THRESHOLDS.fetchCompleteness.toFixed(2)} required).`);
    }
    if (formulaCoverage < GATE_THRESHOLDS.formulaCoverage) {
      blockedConclusions.push(`Formula anti-pattern checks and dependency graph suppressed — formula coverage ${formulaCoverage.toFixed(2)} (minimum ${GATE_THRESHOLDS.formulaCoverage.toFixed(2)} required).`);
    ```

| Boundary | Description |
|----------|-------------|
| Browser → /api/model-state | Untrusted: workspaceId, modelId in body; user/pass in headers |
| /api/model-state → Anaplan API | Token-authenticated server-to-server call |

| Vercel Blob → downstream consumers | URL passed in SSE event; downstream must validate before fetch |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-05 | Spoofing | x-anaplan-user / x-anaplan-pass headers | accept | Anaplan-side authentication is the source of truth; failed auth returns SSE error and aborts before any Blob write |
| T-06-06 | Tampering | workspaceId / modelId in POST body | mitigate | Both lowercased via `toLowerCase()`; embedded in URL only as path segments; no SQL/shell context; missing values return 400 |
| T-06-07 | Repudiation | No request log | accept | No audit requirement in v3.0; Vercel access logs cover request-level tracing |
| T-06-08 | Information Disclosure | Credentials in logs | mitigate | `safeLog` helper strips `authorization`, `password`, `token`, `tokenvalue`, `x-anaplan-user`, `x-anaplan-pass`; acceptance criterion forbids `console.log(token)`, `console.log(password)`, `console.log(req.headers)` |
| T-06-09 | Information Disclosure | Model state Blob is public-read | accept | Vercel Blob URLs are unguessable (random suffix); same disposition as v2.0 blueprints; 7-day TTL via cleanup cron |
| T-06-10 | Denial of Service | maxDuration 60s exceeded by giant model | mitigate | Two parallel calls plus serialization fits well within 60s for 228-module model; for larger models the function will time out and return SSE error rather than producing a partial Blob |
