From api/blueprint.js (auth pattern):
```javascript
const encoded = Buffer.from(`${username}:${password}`).toString('base64');
const authRes = await fetch('https://auth.anaplan.com/token/authenticate', {
  method: 'POST',
  headers: { 'Authorization': `Basic ${encoded}`, 'Content-Type': 'application/json' },
  body: ''
});
const authData = await authRes.json();
const token = authData?.tokenInfo?.tokenValue; // string

// Subsequent calls use:
headers: { 'Authorization': `AnaplanAuthToken ${token}` }
```

- `GET https://api.anaplan.com/2/0/workspaces/{wsId}/models/{modelId}/modules`
- `GET https://api.anaplan.com/2/0/workspaces/{wsId}/models/{modelId}/lineItems?includeAll=true`

`wsId` and `modelId` MUST be lowercased before use (per blueprint.js convention).

    1. Reads `ANAPLAN_USER`, `ANAPLAN_PASS`, `ANAPLAN_WORKSPACE_ID`, `ANAPLAN_MODEL_ID` from `process.env`. Exits with a clear message listing missing vars if any is absent.
    2. Lowercases `ANAPLAN_WORKSPACE_ID` and `ANAPLAN_MODEL_ID`.
    3. POSTs to `https://auth.anaplan.com/token/authenticate` with `Authorization: Basic ${base64(user:pass)}` and empty body. Extracts `tokenInfo.tokenValue`. Exits with the response body printed if `tokenValue` is missing.
    4. Fires `Promise.all([modRes, liRes])` against:
       - `https://api.anaplan.com/2/0/workspaces/${wsId}/models/${modelId}/modules`
       - `https://api.anaplan.com/2/0/workspaces/${wsId}/models/${modelId}/lineItems?includeAll=true`
       Both use `Authorization: AnaplanAuthToken ${token}`.
    5. For each response, logs: HTTP status, response headers (specifically `content-type` and any pagination headers like `link`, `x-pagination-*`), and `JSON.stringify(body, null, 2).slice(0, 8000)` (first 8KB so the terminal stays readable for large models).
    6. For the lineItems response specifically, also logs:
       - `Object.keys(body)` — top-level fields (e.g., `items`, `meta`, `nextPage`)
       - `Object.keys(body.items?.[0] ?? {})` — line item field names (THIS reveals the moduleId field name)
       - `typeof body.items?.[0]?.formula` and the first 200 chars of the first non-null formula (confirms formula presence)
       - `body.meta` or any field whose name contains `page`, `cursor`, `next`, or `total` (pagination signal)
       - The total count: `body.items?.length`
    7. NEVER logs the token, password, or `Authorization` header value. Use a `safeLog(label, obj)` helper that strips these keys before printing.
    8. Writes a NEVER-LOG list (matches blueprint.js `safeLog` convention): `['authorization', 'password', 'token', 'tokenvalue', 'x-anaplan-user', 'x-anaplan-pass']`.

    Run command documented in the file header comment:
