<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client browser → api/connect.js | username+password cross from untrusted browser to server |
| api/connect.js → auth.anaplan.com | Basic Auth credential crosses to external Anaplan auth service over HTTPS |
| api/connect.js → api.anaplan.com | Bearer token crosses to Anaplan workspace API over HTTPS |
| api/models.js ← client headers | x-anaplan-user + x-anaplan-pass cross in HTTP request headers from browser |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | api/connect.js catch block | mitigate | Log only `err.message` (string), never `err` object or `req.body`; enforced in action text and acceptance criteria grep |
| T-02-02 | Information Disclosure | api/models.js catch block | mitigate | Same as T-02-01 — `console.error('Models error:', err.message)` only |
| T-02-03 | Spoofing | api/connect.js Basic Auth construction | mitigate | `Buffer.from(...).toString('base64')` server-side only; never btoa() in browser; encoded string never logged |
| T-02-04 | Elevation of Privilege | Workspace ID case sensitivity | mitigate | `workspaceId.toLowerCase()` applied before all API calls in both connect.js and models.js; acceptance criterion greps for `toLowerCase` |
| T-02-05 | Information Disclosure | CORS wildcard on api/connect.js | accept | Single-origin app; no sensitive data in response beyond workspace names (no credentials returned); Anaplan enforces auth on its end |
| T-02-06 | Denial of Service | api/models.js re-auth on every call | accept | One-shot call per workspace selection — not a polling loop; 10s maxDuration caps server time; Pitfall 3 in RESEARCH.md |
| T-02-07 | Spoofing | Invalid credentials response — 401 from Anaplan | mitigate | Any non-2xx from auth endpoint treated as "Invalid credentials"; do not parse error body shape (undocumented by Anaplan) |
</threat_model>
