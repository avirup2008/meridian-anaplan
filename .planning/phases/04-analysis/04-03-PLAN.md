    **Pre-flight (5 min):**
    1. Confirm `ANTHROPIC_API_KEY` is set in Vercel dashboard (Vercel → Project → Settings → Environment Variables).
    2. Confirm `BLOB_READ_WRITE_TOKEN` is set (needed for Phase 3 blueprint write).
    3. Deploy current main: `vercel --prod` (or `git push` if auto-deploy enabled).
    4. Open https://{your-meridian-url}/ in a fresh incognito window.

    **Walk-through (10 min):**
    5. Click through s-intro → s-connect → enter Anaplan credentials.
    6. On s-picker, choose the **COPS Demo** model (or any model with > 50 modules).
    7. On s-fetch, click "Fetch Blueprint" and wait for progress to complete. Confirm `meridian.blueprintBlobUrl` is set in DevTools → Application → sessionStorage.
    8. Click "Continue" → app navigates to s-analysis. The runAnalysis() call should fire automatically.
    9. **ANLZ-03 check (extraction):** Within the first 2-3 seconds, the progress bar should advance past "Extracting modules" with a detail line like "N modules, M line items extracted." Pull Vercel function logs: search for "Token budget" or `countTokens` evidence; confirm no token-limit error.
    10. **ANLZ-02 check (Haiku suggestions):** Progress bar should advance through "Analysing modules (Haiku)" with live module names ticking by. Each suggestions panel card must show a triage pill in red/gold/blue (Fix Now / Consider / Monitor) and a domain header (Structural / Formula / Best Practice / Naming).
    11. **ANLZ-01 check (Sonnet score):** Score card appears with: a number 0-100, a verdict chip ("Good", "Needs Work", or "Critical"), an executive summary paragraph (2-3 sentences), and five dimension bars (Architecture, Naming, Formulas, Data Hygiene, Governance) each showing a numeric value and filled bar.
    12. **ANLZ-04 check (narrative + drill-in):** Below the suggestions, a Cross-Module Data Flow story renders with paragraphs of text, followed by clickable module chips. Click any chip → drill-in panel appears with "← Back to story" breadcrumb, the module's purpose, and three lists: Receives from / Sends to / Risks. Click "Back to story" → story view returns.
    13. **Timing check:** Open DevTools → Network → click the `/api/analyze` request → confirm it completed (status 200) and "Time" is < 60s.
    14. **Partial-analysis path (optional but valuable):** If your test model has > 150 modules, you may see a "Partial analysis" yellow banner. Confirm the analysis still completes (score card still renders) — partial is non-fatal.
    15. **Security checks:** Open DevTools → Console. Confirm:
        - No log line contains `ANTHROPIC_API_KEY`, `x-anaplan-pass`, or the literal password.
        - No log line contains the full blobUrl.
        - Vercel function logs (separate tab): no `req.body` dumps, no Stack traces with credentials.

    **Document results:** Create `/tmp/meridian-anaplan/.planning/phases/04-analysis/04-SIGNOFF.md` with:
    - Date + Vercel deployment URL + commit SHA
    - Model name + module count + line item count
    - Observed timings: extraction (s), Haiku batches total (s), Sonnet synthesis (s), Sonnet narrative (s), grand total (s)
    - healthScore + verdict values seen
    - Suggestion count by domain and by triage
    - Anaplan-builder gut-check on triage calibration (per STATE.md research flag: "Triage tag calibration needs review with Anaplan builder before Phase 4 ships")
    - Screenshots (optional): score card, suggestions panel, drill-in
    - Any deviations from the spec; any follow-ups to defer
